### When to Cite the Preprint Manuscript

**Use this citation when discussing**:
- Theoretical framework and physics claims
- Validation methodology (7-tier approach)
- Overall results and conclusions
- Scientific discoveries (emergent gravity, quantum, EM, thermodynamics)

**Citation**: Partin, G. D. (2025). Emergent gravity... OSF Preprints. doi: 10.17605/OSF.IO/6AGN8

### When to Cite the Software

**Use this citation when**:
- Using the LFM code in your own research
- Extending or modifying the codebase
- Benchmarking against LFM results
- Describing the computational implementation

**Citation**: Partin, G. D. (2025). Lattice Field Medium v3.2 [Software]. Zenodo. doi: 10.5281/zenodo.17536484

### When to Cite the Dataset

**Use this citation when**:
- Analyzing LFM validation data
- Comparing your results to LFM benchmarks
- Reproducing LFM experiments
- Meta-analysis of computational physics validation

**Citation**: Partin, G. D. (2025). LFM Phase 1 validation dataset [Data set]. Zenodo. doi: 10.5281/zenodo.17536484

## QUESTIONS ABOUT CITATIONS?

### Common Questions

**Q: Can I cite just the GitHub repository?**  
A: No. Always cite the DOI (Zenodo or OSF). GitHub links can break; DOIs are permanent.

**Q: Do I need to cite if I just read the work but don't use it?**  
A: If it influenced your thinking or approach, yes. Citation acknowledges intellectual debt.

**Q: What if I only use one tier of tests?**  
A: Cite the full preprint, then specify in-text which tier you used.

**Q: Can I cite this before it's peer-reviewed?**  
A: Yes. It's a preprint with DOI. Just note "preprint" or "not peer-reviewed" if required.

**Q: What if I modify the code?**  
A: Still cite the original software. Add your own DOI for your modified version.
